java-spi

发表于 2020-06-07 分类于 java

是什么？

SPI，全称Service Provider Interface，是Java提供的一套用来被第三方实现或者扩展的API，它可以用来启用框架扩展和替换组件。

优点

  1. 解耦，直接替换实现jar包即可，client端loader是接口，无需做任何更改，比如jdbc的驱动。
  2. 能跨jar包加载，即实现jar在自己的META-INF/services目录创建即可，无需使用方创建，可以参考mysql-connector-driver。

缺点

  1. 只能通过遍历全部获取，即实现类全部加载并实例化。并不想用的实现类，也被加载并实例化，造成浪费。
  2. 获取某个实现类的方式不够灵活，只能通过Iterator形式获取，不能根据某个参数来获取对应的实现类。
  3. 多个并发多线程使用ServiceLoader类的实例是不安全的。

使用步骤

  1. 写一个接口；
  2. 写几个实现类；
  3. 创建目录和文件：
    • 在resources目录下创建META-INF/services目录
    • 创建和接口名相同的文件，放到service目录下；
    • 在文件中加入实现类的全类名；
  4. 在代码中使用ServiceLoader.loader(接口名.class)加载，就会加载文件中的所有实现类；

使用约束

要使用Java SPI，需要遵循如下约定：

  • 当服务提供者提供了接口的一种具体实现后，在jar包的META-INF/services目录下创建一个以“接口全限定名”为命名的文件，内容为实现类的全限定名；
  • 接口实现类所在的jar包放在主程序的classpath中；
  • 主程序通过java.util.ServiceLoder动态装载实现模块，它通过扫描META-INF/services目录下的配置文件找到实现类的全限定名，把类加载到JVM；
  • SPI的实现类必须有无参构造器；
