protobuf-概览

发表于 2020-06-05 更新于 2020-06-06 分类于 protobuf

是什么？

2个人通信的时候，需要用相同的信息格式。比如一句话完了之后用什么符号来标识，如果收信的一方不知道这个规则，就会看不懂。所以需要约定一个格式。

Protocol Buffer 就是制定了一种格式，如果通信双方都使用这种格式，那就能看懂对方发来的信息。

能干什么？

Protocol Buffer 不止制定了一种格式，而且提供了一套工具，各种语言的都有。

由哪些部分组成？

  • 原始数据，
  • 原始数据的描述文件.proto类似于xml的xsd文件，
  • 编解码工具。

分布式-RPC

发表于 2020-06-04 分类于 分布式

是什么？

RPC（Remote Procedure Call）远程过程调用，是一种进程通信方式，是一种技术思想，不是规范。允许进程调用另一个地址空间（通常是共享网络的另一台机器上）的过程或函数，而不用程序员显式编码这个远程调用的细节。即程序员无论是调用本地的还是远程的函数，本质上编写的调用代码基本相同。

基本原理

流程

一次完整的RPC调用流程（同步调用，异步另说）如下：

  1. 服务消费方（client）调用以本地调用方式调用服务；
  2. client stub接收到调用后负责将方法、参数等组装成能够进行网络传输的消息体；
  3. client stub找到服务地址，并将消息发送到服务端；
  4. server stub收到消息后进行解码；
  5. server stub根据解码结果调用本地的服务；
  6. 本地服务执行并将结果返回给server stub；
  7. server stub将返回结果打包成消息并发送至消费方；
  8. client stub接收到消息，并进行解码；
  9. 服务消费方得到最终结果。

RPC框架的目标就是要2 ~ 8这些步骤都封装起来，这些细节对用户来说是透明的，不可见的。

性能主要影响点

  1. 网络通信传输效率（传输的数据量大小）；
  2. 解析（序列化和反序列化）效率；

常见框架

dubbo、gRPC、Thrift、HSF（High Speed Service Framework）
